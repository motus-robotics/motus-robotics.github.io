Motus2: Advancing Dexterous Manipulation to Human Level

Motus2 is a self-evolved egocentric embodied agent built on a unified world model, scaling human-centric data from web video to tactile robot demonstrations — on a bionic stereo dual-arm platform with full-palm tactile sensing.

Unified World Model × Human Data × Dextrous Hand × Tactile Glove × Self-Evolving = The End

TL;DR

  • Manipulation is the core bottleneck — contact-rich, long-horizon dexterity requires full-stack thinking beyond model training alone.
  • Human-centric data at scale — a five-tier data pyramid from web video (110k+ hrs ego) to tactile robot demos bridges the embodiment gap.
  • Joint Video-Action-Tactile DiT — multi-token in, multi-token out; one model for control, simulation, IDM, and value estimation.
  • Self-evolving via MBRL — Ray-based distributed async RL turns failure and suboptimal data into continuous improvement.
  • Predictable scaling — human data scaling laws show consistent loss reduction as data grows from 2k to 20k hours.

Motivation

Manipulation: why, and how.

Manipulation is the most valuable problem in robotics because it turns intelligence into useful work. Most physical labor is not about moving through the world, but transforming it — and that transformation happens through contact.

Motus2 addresses this as a full-stack system problem, not a pure model training problem. Hardware that can express rich contact, data that preserves human interaction fidelity, a robotics-native foundation model, and evaluation infrastructure that scales — all must work together.

Our robot platform combines stereo egocentric vision, a bionic dual-arm dual-hand system, and full-palm tactile sensing — designed to minimize the embodiment gap between human demonstrations and robot execution.

Architecture

A robotics-native foundation model

Motus2 learns a joint distribution over video, action, and tactile trajectories using a Joint Video-Action-Tactile DiT. Multi-token input and multi-token output enable a single unified model for control, generative simulation (AC-WM), inverse dynamics, and value estimation — with missing modalities inferred through denoising.

Post-training supports four modes: joint, video_idm, action_only, and loop. A sliding-window autoregressive scheme enables long-horizon rollouts at deployment.

Scaling Path

The human-centric data engine

Over 80% of physical labor is manipulation — almost none of it has ever been recorded. Motus2 is built around a scaling path from diverse human demonstrations to tactile robot data, spanning five tiers of the data pyramid.

Human Data Scaling Laws

Increasing human data consistently reduces validation loss. From 2k to 20k hours, models converge faster and reach lower asymptotic error — following a predictable logarithmic scaling law.

L = 0.101 − 0.005 · ln(D)

Self-Evolving

Scaling towards deployment

Beyond imitation learning, Motus2 leverages failure data, suboptimal trajectories, and task-irrelevant data through a self-evolving pipeline powered by Ray-based distributed asynchronous RL — continuously improving policy quality from real-world feedback.

Test-Time Scaling

At inference, Motus2 performs iterative denoising via IDM + FDM, alternating between action prediction and video rollout for improved planning:

1

Predict Action

Roll out one action step

2

Simulate Video

FDM / AC-WM generates observation

3

Infer Action

IDM / Policy refines action

4

Loop

Repeat until completion

Evaluation

Results & ablation

Each component of the Motus2 stack contributes measurably to downstream task performance. On screw-bulb insertion with only 100 teleop demos, the full pipeline lifts success rate from 5% to 90%.
